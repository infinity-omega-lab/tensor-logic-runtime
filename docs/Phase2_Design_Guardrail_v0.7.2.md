# Phase 2 Design Guardrail
**Tensor Logic Runtime v0.7.2 — Non-Execution Design Boundary**

---

## 0. Status Declaration（最初に固定する事実）

- TL Runtime **v0.7.2 は正しさの基準点であり freeze 済み**
- 本文書は **Phase 2 を開始するための設計ではない**
- 本文書の目的は  
  **「Phase 2 において“やってはいけないこと”を明示すること」** にある

本フェーズでは、  
**runtime execution / reduce semantics / reference behavior に一切の変更を加えない。**

---

## 1. Phase 2 の目的（限定定義）

Phase 2 の目的は以下に**厳密に限定**される：

- `#pragma` 構文が **runtime に存在を通知できる最小の概念接点**を設計する
- それ以上の意味・挙動・評価を **一切与えない**
- 成果物は **設計文書（Markdown）のみ**

> Phase 2 は「接続しない接続点」を定義するフェーズである。

---

## 2. 明示的に禁止される行為（Non-Goals）

Phase 2 において **以下はすべて禁止**とする：

- ReduceExecutor / ReducePlan / DomainBinder への変更
- runtime execution フローへの介入
- pragma による意味解釈・条件分岐・最適化
- 暗黙推論・自動評価・自動選択の導入
- コード実装（例外なし）
- テスト追加・変更

> 「将来できそう」は許可されるが  
> **「今できる」はすべて禁止**とする。

---

## 3. 許可される設計範囲（Hard Boundary）

Phase 2 で許可されるのは以下のみ：

- pragma の **構文的存在**
- runtime が pragma を **観測可能であるという事実の定義**
- その観測が **実行結果に影響しない**ことの明文化
- 「将来どこに繋がるか」を *参考情報として* 記述すること

必須条件：

- 「現時点では何も起こらない」ことを必ず併記する
- 実行経路に影響しないことを明文化する

---

## 4. Phase 2 強制停止チェックリスト

以下のいずれかに **YES** が出た場合、  
Phase 2 の作業は **即時中断**する。

- 書いていてコードを触りたくなっていないか
- Reduce の意味を再定義しようとしていないか
- runtime execution に影響が出る表現になっていないか
- pragma に「判断」や「選択」をさせようとしていないか

> Phase 2 は「安全に止まれるか」を試すフェーズである。

---

## 5. 終了条件（Done Definition）

Phase 2 の第一段は、以下を満たした時点で完了とする：

- 「やらないこと」が明確に書かれている
- 実装に進める余地が残っていても、**今は進まないと判断できる**
- v0.7.2 の reference runtime に一切の疑念が生じていない
